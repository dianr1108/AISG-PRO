1. LATAR BELAKANG & TUJUAN SISTEM
================================================================================

AISG (Audit Intelligence SG) adalah platform audit performa karyawan berbasis
korporat enterprise yang dirancang untuk mengevaluasi performa karyawan dan
leadership menggunakan framework standar 18 Pilar.

MENGAPA AISG DIBUTUHKAN?
------------------------
a) Evaluasi Subjektif Manual:
   - Sebelumnya, audit performa dilakukan manual menggunakan spreadsheet
   - Tidak ada standarisasi scoring dan penilaian
   - Sulit tracking historical performance quarter-to-quarter
   - Rekomendasi ProDem (Promotion-Demotion) tidak data-driven

b) Kompleksitas Framework 18 Pilar:
   - Framework 18 Pilar memiliki aturan scoring yang kompleks
   - Perhitungan Reality Score, Behavioral Zone, dan Performance Zone 
     memerlukan algoritma yang konsisten
   - Manual calculation prone to human error

c) Kurangnya Insight Personalisasi:
   - Karyawan tidak mendapat feedback yang actionable
   - Tidak ada coaching atau motivasi yang personalized
   - Sulit identifikasi gap antara self-assessment vs expected standard

TUJUAN SISTEM AISG:
-------------------
1. STANDARDISASI: Menyediakan framework audit yang konsisten dan terukur
2. AUTOMATION: Mengotomasi perhitungan kompleks (Reality Score, Zona, ProDem)
3. TRANSPARENCY: Memberikan visibility penuh kepada karyawan tentang hasil audit
4. ACTIONABLE INSIGHTS: Menyediakan AI Coach untuk personalized guidance
5. DATA-DRIVEN DECISIONS: Membantu manajemen membuat keputusan ProDem yang objektif
6. QUARTERLY TRACKING: Monitoring performa kuartalan (Q1-Q4) untuk trend analysis

STAKEHOLDER UTAMA:
------------------
- Karyawan: Business Manager (BsM), Sales Business Manager (SBM), etc.
- Auditor: Tim HR/Performance Management yang melakukan evaluasi
- Manajemen: Leadership yang menggunakan data untuk strategic decisions

2.1 MULTI-STEP AUDIT FORM (5 LANGKAH)
--------------------------------------

STEP 1: INFORMASI PERSONAL
- Input: Nama, Jabatan, Cabang, Tanggal Lahir
- Validasi: Semua field wajib diisi
- Jabatan options: Business Manager (BsM), Sales Business Manager (SBM), dll.

STEP 2: PERFORMA QUARTERLY (Q1-Q4)
- Input Data Tim:
  * Margin Tim Q1, Q2, Q3, Q4 (dalam Rupiah)
  * New Account (NA) Tim Q1, Q2, Q3, Q4 (jumlah nasabah baru)
  
- Input Data Pribadi (NEW!):
  * Margin Pribadi Q1, Q2, Q3, Q4 (kontribusi personal)
  * Nasabah Pribadi Q1, Q2, Q3, Q4 (akuisisi personal)

- Fitur UX:
  * Input number yang clean (tidak ada bug "010000")
  * Value 0 ditampilkan sebagai placeholder kosong
  * Format Rupiah otomatis dengan separator

STEP 3: STRUKTUR TIM (TEAM HIERARCHY)
- Input jumlah:
  * Business Consultant (BC)
  * Senior Business Consultant (SBC)
  * Business Manager (BsM)
  * Senior Business Manager (SBM)
  * Executive Manager (EM)
  * Senior Executive Manager (SEM)
  * Vice Business Manager (VBM)

- Validasi: Minimal 1 team member harus diisi

  * 5 = Sangat Baik

  1. Integritas & Etika Kerja

  4. Problem Solving & Decision Making

  8. Pengetahuan Produk
  9. Customer Service Excellence
  10. Sales & Negotiation Skills
  11. Target Achievement & KPI
  12. Financial & Business Acumen
  13. Coaching & Mentoring

  16. Innovation & Continuous Improvement

STEP 5: REVIEW & SUBMIT

- Submit → Generate full audit report + redirect ke detail page

2.2 AUDIT REPORT (12 SECTION COMPREHENSIVE)
--------------------------------------------

Section 1: EXECUTIVE SUMMARY
- Overview singkat hasil audit
- Reality Score total (max 90)
- Zona Final (Success/Warning/Critical)
- Profil Karyawan (Consistency/Balance/Zone-based)

Section 2: REALITY SCORE CALCULATOR
- Total score dari 18 Pilar self-assessment
- Perbandingan vs expected score
- Visual indicator zona performance

Section 3: PERFORMANCE ZONE
- Analisis berdasarkan margin & NA quarterly
- Zona: Success (🟩), Warning (🟨), atau Critical (🟥)
- Trend quarterly performance

Section 4: BEHAVIORAL ZONE
- Analisis pola behavioral berdasarkan 18 Pilar
- Identifikasi strength & weakness areas
- Rekomendasi behavioral improvement

Section 5: PROFIL KARYAWAN
- Tipe profil: Consistency Expert, Balanced Performer, dll.
- Karakteristik profil detail
- Strength & area for development

Section 6: SWOT ANALYSIS
- Strengths: Kelebihan berdasarkan data
- Weaknesses: Area yang perlu improvement
- Opportunities: Peluang growth
- Threats: Risk factors yang perlu diwaspadai

Section 7: EARLY WARNING SYSTEM (EWS)
- Identifikasi red flags dalam performa
- Risk level assessment
- Preventive action recommendations

Section 8: ACTION PLAN 30-60-90
- 30 hari pertama: Quick wins & foundational changes
- 60 hari: Skill development & process improvement
- 90 hari: Strategic initiatives & long-term goals

Section 9: PRODEM RECOMMENDATION
- Recommendation: Promosi, Demotion, atau Hold
- Reasoning: Data-driven justification
- Strategy: Save by Margin vs Save by Staff rotation
- Timeline: Kapan evaluasi ulang dilakukan

Section 10: MAGIC SECTION (PERSONALIZED MOTIVATION)
- Zodiac-based motivational quote
- Personalized booster message
- Gamification element untuk engagement

Section 11-12: ADDITIONAL INSIGHTS
- Team dynamics analysis
- Quarterly trend comparison
- Historical performance tracking (jika ada)

BACKEND:
- Runtime: Node.js dengan Express.js
- Language: TypeScript
- Validation: Zod schemas (shared dengan frontend)
- API Design: RESTful JSON API

DATABASE:
- PostgreSQL (Neon serverless)
- ORM: Drizzle ORM
- Migrations: Drizzle Kit
- Connection: @neondatabase/serverless dengan pooling

AI INTEGRATION:
- Provider: OpenAI GPT-4o
- Library: openai npm package
- Model: gpt-4o (most capable omni model)
- Max Tokens: 500 per response
- Temperature: 0.7 (balanced creativity)

3.2 DATABASE SCHEMA
-------------------

TABLE: users
- id (UUID primary key)
- username (varchar, unique)
- password (varchar, hashed)
- email (varchar, unique)
- role (varchar: 'auditor', 'manager', 'admin')
- createdAt (timestamp)

TABLE: branches
- id (UUID primary key)
- name (varchar)
- region (varchar)
- createdAt (timestamp)

TABLE: audits
- id (UUID primary key)
- nama (varchar) - Employee name
- jabatan (varchar) - Position
- cabang (varchar) - Branch
- tanggalLahir (varchar) - Birth date
- marginTimQ1, Q2, Q3, Q4 (integer) - Team margin quarterly
- naTimQ1, Q2, Q3, Q4 (integer) - Team NA quarterly
- marginPribadiQ1, Q2, Q3, Q4 (integer) - Personal margin quarterly (NEW!)
- nasabahPribadiQ1, Q2, Q3, Q4 (integer) - Personal NA quarterly (NEW!)
- jumlahBC, SBC, BsM, SBM, EM, SEM, VBM (integer) - Team structure
- pillarAnswers (jsonb) - 18 Pilar self-assessment scores
- totalRealityScore (integer) - Calculated total score
- zonaPerforma (varchar) - Performance zone
- zonaBehavioral (varchar) - Behavioral zone
- zonaFinal (varchar) - Final zone
- profil (varchar) - Employee profile type
- magicSection (jsonb) - Personalized motivation data
- prodemRekomendasi (jsonb) - ProDem recommendation details
- createdAt (timestamp)

TABLE: chatMessages
- id (UUID primary key)
- auditId (UUID, foreign key CASCADE to audits)
- role (varchar: 'user' or 'assistant')
- content (text)
- createdAt (timestamp)

INDEXES:
- audits.nama (for search by name)
- audits.createdAt (for sorting)
- chatMessages.auditId (for chat history queries)

3.3 API ENDPOINTS
-----------------

POST /api/audit
- Body: Full audit data (personal info + quarterly + team + 18 pilar)
- Returns: Calculated audit result dengan all 12 sections
- Processing: Calls business-logic.ts untuk perhitungan kompleks

GET /api/audits
- Returns: Array of all audits (list view)
- Sorting: By createdAt DESC (newest first)

GET /api/audit/:id
- Returns: Single audit detail by ID
- 404 if not found

GET /api/audits/name/:name
- Returns: Array of audits filtered by employee name
- Case-insensitive search

DELETE /api/audit/:id
- Cascade delete: Hapus chat history dulu, lalu audit
- Returns: Success message

POST /api/chat
- Body: { auditId, message }
- Processing:
  1. Save user message to DB
  2. Load chat history (last 10 messages)
  3. Build system prompt with audit context
  4. Call OpenAI GPT-4o API
  5. Save AI response to DB
  6. Return AI response
- Error Handling: 429 for rate limit, 500 for general error

GET /api/chat/:auditId
- Returns: Array of chat messages for audit
- Ordered by createdAt ASC (oldest first)

DELETE /api/chat/:auditId
- Clear all chat history for audit
- Returns: Success message

GET /api/audit/:id/pdf
- Generates PDF on-the-fly using pdfkit
- Returns: PDF file stream
- Content-Disposition: attachment (auto download)

3.4 BUSINESS LOGIC (CORE ALGORITHMS)
-------------------------------------

FILE: server/business-logic.ts

FUNCTION: processAudit(data)
- Input: Raw audit form data
- Output: Processed audit dengan calculated fields

STEPS:
1. Calculate Total Reality Score (sum of 18 pilar scores)
2. Determine Performance Zone:
   - Aggregate quarterly margin & NA
   - Compare vs thresholds
   - Assign: Success (🟩), Warning (🟨), Critical (🟥)

3. Determine Behavioral Zone:
   - Analyze 18 pilar score distribution
   - Identify patterns (consistent, improving, declining)
   - Assign behavioral zona

4. Determine Final Zone:
   - Weighted combination of performance + behavioral
   - Final decision: Success/Warning/Critical

5. Generate Employee Profile:
   - Pattern recognition dari quarterly data
   - Types: Consistency Expert, Balanced Performer, Rising Star, dll.
   - Based on: variance, trend, zone distribution

6. Generate SWOT Analysis:
   - Strengths: Top 3 pilar scores + zona success areas
   - Weaknesses: Bottom 3 pilar scores + zona critical areas
   - Opportunities: Growth potential based on trend
   - Threats: Risk factors dari EWS

7. Generate ProDem Recommendation:
   - Algorithm:
     * IF zona = Success + trending up → PROMOSI
     * IF zona = Critical + trending down → DEMOTION
     * ELSE → HOLD (with improvement plan)
   - Strategy: Save by Margin vs Save by Staff (rotation)
   - Timeline: 1-3 months evaluation period

8. Generate Magic Section:
   - Zodiac calculation dari tanggal lahir
   - Load motivational quote based on zodiac + profil
   - Personalized booster message
   - Gamification elements

9. Generate Action Plan 30-60-90:
   - 30 days: Quick wins (low-hanging fruits)
   - 60 days: Skill development (training, coaching)
   - 90 days: Strategic initiatives (process changes)
   - Prioritized based on weaknesses & opportunities

10. Generate EWS (Early Warning System):
    - Red flags detection:
      * Declining quarterly trend (3+ quarters)
      * Reality Score < 50% of max
      * Critical zona for 2+ consecutive quarters
      * High variance in team structure
    - Risk level: Low/Medium/High
    - Preventive actions recommended

6. KEKURANGAN & KETERBATASAN
================================================================================

6.1 KETERBATASAN TEKNIS
-----------------------

a) DEPENDENCY PADA INTERNET
   ❌ System 100% web-based → butuh internet connection
   ❌ Tidak ada offline mode
   ❌ Jika internet down, sistem tidak bisa diakses
   
   Mitigasi: Deploy di cloud reliable (Replit) dengan 99.9% uptime

b) AI QUOTA LIMITATION
   ❌ OpenAI API ada quota limit → bisa error jika quota habis
   ❌ Butuh top-up API credit ($5-10/month tergantung usage)
   ❌ ChatGPT Plus subscription tidak cover API usage
   
   Mitigasi: 
   - Monitor usage via OpenAI dashboard
   - Set up billing alerts
   - Alternative: Switch ke Google Gemini (free tier generous)

c) SINGLE DATABASE INSTANCE
   ❌ Hanya 1 PostgreSQL database (no backup/replica automatic)
   ❌ Jika database corrupt, data bisa hilang
   ❌ No disaster recovery plan built-in
   
   Mitigasi:
   - Neon PostgreSQL has automatic backups
   - Consider setting up manual backup cron job
   - Export critical data regularly

d) NO REAL-TIME COLLABORATION
   ❌ Tidak ada multi-user edit audit bersamaan
   ❌ No version control untuk audit (satu audit = final result)
   ❌ No commenting/approval workflow
   
   Future Enhancement: Add audit workflow (draft → review → approved)

e) LIMITED AUTHENTICATION
   ❌ Belum ada proper user authentication & authorization
   ❌ Semua user bisa lihat semua audit (no access control)
   ❌ No role-based permissions (auditor vs manager vs karyawan)
   
   Future Enhancement: Implement Replit Auth atau session-based auth

f) MOBILE RESPONSIVENESS
   ❌ UI optimized untuk desktop, tapi mobile experience bisa lebih baik
   ❌ Chat panel di mobile agak cramped
   ❌ PDF generation di mobile sometimes tricky
   
   Future Enhancement: Native mobile app atau Progressive Web App (PWA)

6.2 KETERBATASAN BISNIS
------------------------

a) MANUAL DATA ENTRY
   ❌ Semua data harus diisi manual (tidak ada integration dengan HRIS)
   ❌ Prone to human error saat input
   ❌ Time-consuming jika audit banyak karyawan sekaligus

8. TROUBLESHOOTING
================================================================================

8.1 MASALAH UMUM & SOLUSI
-------------------------

MASALAH 1: "Cannot submit audit form"
PENYEBAB:
- Form validation error (ada field yang missing)
- Network error (internet connection issue)

SOLUSI:
1. Check semua field step 1-4 sudah diisi
2. Buka browser console (F12) → lihat error message
3. Refresh page dan coba lagi
4. Jika masih error, clear browser cache

---

MASALAH 2: "AI Chat tidak respond / loading terus"
PENYEBAB:
- OpenAI API quota habis (most common!)
- Network timeout
- Server error

SOLUSI:
1. Tunggu 10-15 detik (OpenAI bisa slow)
2. Check error toast message:
   - "Layanan AI sedang mencapai batas penggunaan" → Quota habis
   - "Terjadi kesalahan saat menghubungi AI" → Server error
3. Jika quota habis:
   - Login ke https://platform.openai.com
   - Go to Billing → Add payment method → Top up $5-10
4. User message tetap tersimpan, coba send lagi setelah top up

---

MASALAH 3: "PDF download tidak jalan"
PENYEBAB:
- Browser block popup/download
- Server error saat generate PDF

SOLUSI:
1. Check browser popup blocker → allow popups dari site
2. Check browser download settings → allow automatic downloads
3. Try different browser (Chrome recommended)
4. Check browser console untuk error message

---

MASALAH 4: "Input number jadi '010000' pas diisi"
STATUS: FIXED ✅
PENYEBAB: Bug di input value handling

SOLUSI:
- Sudah diperbaiki dengan conditional rendering:
  value={field.value === 0 ? "" : field.value}
- Update code ke versi terbaru

---

MASALAH 5: "Audit list tidak muncul / kosong"
PENYEBAB:
- Belum ada audit yang dibuat
- Database connection error
- Query error

SOLUSI:
1. Buat audit baru untuk populate data
2. Check browser console → lihat API error
3. Refresh page
4. Check database connection (admin only)

---

MASALAH 6: "Delete audit tidak jalan"
PENYEBAB:
- Confirmation dialog tidak di-confirm
- Database cascade delete error

SOLUSI:
1. Pastikan klik "Delete" di confirmation dialog (bukan Cancel)
2. Wait for success toast message
3. Refresh page jika audit masih muncul
4. Check browser console untuk error

---

MASALAH 7: "Website slow / loading lama"
PENYEBAB:
- Replit free tier (sleep after inactivity)
- Cold start delay (~5-10 detik)
- Large database query

SOLUSI:
1. First load slow = normal (cold start)
2. Subsequent loads harusnya fast
3. Upgrade ke Replit paid plan untuk always-on
4. Optimize database queries (add indexes)

---

MASALAH 8: "Dark mode tidak apply"
PENYEBAB:
- Browser cache issue
- Theme setting tidak save

SOLUSI:
1. Hard refresh: Ctrl+Shift+R (Windows) atau Cmd+Shift+R (Mac)
2. Clear browser cache
3. Check localStorage untuk theme setting
4. Toggle theme di theme switcher (jika ada)

8.2 ERROR CODES & MEANINGS
---------------------------

HTTP 400 - Bad Request
- Form validation error
- Invalid input data
- Fix: Check form fields, ensure correct data types

HTTP 404 - Not Found
- Audit ID tidak ditemukan
- Route tidak exist
- Fix: Check URL, ensure audit exists

HTTP 429 - Rate Limit
- OpenAI API quota exceeded
- Too many requests
- Fix: Top up API credit, wait dan retry

HTTP 500 - Internal Server Error
- Server crash
- Database error
- Uncaught exception
- Fix: Check server logs, contact admin

8.3 DEBUGGING TIPS
------------------

UNTUK USER:
1. Browser Console (F12) adalah teman Anda!
   - Tab Console: Lihat JavaScript errors
   - Tab Network: Lihat API requests (status code, response)
   - Tab Application: Check localStorage, cookies

2. Screenshot Error Message
   - Include full screen (URL visible)
   - Include error toast/dialog
   - Send ke admin untuk investigation

3. Try Different Browser
   - Chrome (recommended)
   - Firefox
   - Safari
   - Edge
   - Avoid IE (not supported!)

4. Incognito Mode
   - Test di incognito untuk rule out cache/extension issues

UNTUK DEVELOPER:
1. Server Logs
   - Check console output di Replit
   - Look for "[CHAT]", "[ERROR]", "[DB]" prefixes
   - Stack traces untuk debugging

2. Database Inspection
   - Use Replit DB viewer
   - Run SQL queries untuk check data integrity
   - Verify migrations applied correctly

3. Network Tab
   - Check API request payload & response
   - Verify status codes
   - Look for failed requests

4. React Query Devtools (if installed)
   - Inspect cache state
   - See query loading/error states
   - Debug refetch behavior
